# 安全策略

## 支持范围

安全修复优先提供给最新发布版本；公开注册表能力在完成威胁模型和安全门禁前不会标记为稳定。

## 报告漏洞

请不要为未修复的供应链、安全或凭据问题创建公开 Issue。使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告功能，并提供受影响版本与系统、最小复现、可观察影响和已知缓解方式。

涉及言序核心解析器的漏洞会与`YanXuLang/yanxu`协调修复和披露。言包 0.2.1 要求言序 1.1.6 安全基线；远程制品中的链接、设备文件、FIFO、越界路径和超限归档会在进入缓存前被拒绝，工程子进程也受权限、超时、UTF-8 与输出大小门禁约束。请在公开披露前为修复预留合理时间。
