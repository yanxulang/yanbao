# 言包 0.2 架构

## 纯言序命令层

`src/主.yx`实现命令解析、工作流、回滚和人类输出，`src/核心.yx`封装进程调用及工程协议。三个跨平台启动器只定位 `yanxu` 与入口文卷，不解析参数、不修改清单，也不包含依赖业务。

言包要求用户已经安装言序。这一约束是有意设计：言包不再携带一套 Rust 运行时，也不需要 Rust 工具链；言序运行器同时提供执行环境和唯一包核心。

## 唯一依赖语义

言包通过 `yanxu package protocol <JSON>`调用协议版本 1。协议握手公开言序版本、目标平台以及支持的清单、锁、字节码、YXB 和原生 ABI 版本。以下能力只在言序核心实现：

- 格式 1/2 清单读取与格式 2 写入；
- 依赖别名、开发依赖、公开导出和包子模块；
- 直接及传递依赖解析、身份、锁定和离线校验；
- YXB 与独立应用构建；
- 原生制品平台、ABI、校验和和权限门禁。

言包不得自行解析 TOML、选择版本、读取注册表索引或拼装锁文件。

## 事务与失败语义

`add/remove`先保存清单和锁文件原文，再请求核心编辑和解析。任一步失败都恢复两份文件；成功后才输出完成信息。工程协议错误是结构化 JSON，言包把错误转换为言序异常，因此进程退出码保持非零。

`标准:进程`受 `进程`权限控制，具备超时、UTF-8 校验和输出大小限制。源码启动器按用户明确调用在宿主权限下运行；若将言包编译为 YXB，权限来自 `言序.toml`。

## 安全边界

- 远程制品解压、安全路径、校验和与压缩上限只由核心处理；
- 依赖只能沿预解析图访问，根包不能直接引用未声明的传递依赖；
- 原生扩展默认禁止，须显式权限和精确目标制品；
- 启动器不执行安装钩子或第三方构建脚本；
- 未经安全评审的公开注册表写入不属于 0.2 发布范围。
