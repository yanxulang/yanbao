# 言包稳定化路线图

本文件记录言包已经完成和正在交付的版本边界。已发布行为以变更记录为准，架构与安全边界见[`ARCHITECTURE.md`](ARCHITECTURE.md)。

## 0.5.1：工具链兼容修复（已完成）

- [x] 最低兼容通道固定使用清单声明的最低稳定言序标签；
- [x] 当前稳定通道动态安装最新正式言序 Release；
- [x] 开发联调通道构建言序`main`并按工程协议能力验收；
- [x] 当前稳定版继续执行六目标独立应用、言窗工程和 Bundle 验收；
- [x] 同步版本元数据与变更记录，并接入标签、Release 和安装复验门禁。

## 0.5.0：言序 1.1.7 GUI 工程（已完成）

- [x] 通过共享工程协议生成`yanxu-gui`项目模板、应用元数据和独立 GUI 权限；
- [x] 同时握手 ABI v1/v2，锁定并校验当前目标原生后端；
- [x] `build --bundle`生成 macOS `.app`、Windows GUI 目录或 Linux AppDir；
- [x] 六目标原生运行器分别构建言窗后端并执行 GUI 工程、Bundle 与摘要门禁。

## 0.6.0：CLI、事务与结构化接口（已完成）

- [x] 严格解析子命令、位置参数、选项值、冲突、`--`透传与未知参数；
- [x] `new/init`、依赖编辑和锁文件更新采用可恢复事务；
- [x] 支持分操作超时、中断转发、清理和稳定退出码；
- [x] 提供 human、JSON 与 JSON Lines 输出及正式 Schema；
- [x] `audit`按严重度、抑制原因和到期时间决定退出状态。

## 0.6.1：安全与供应链（当前发布线）

- [x] 增加依赖漏洞、许可证、来源、校验和、精确修订和原生 ABI 审计；
- [x] 对审计能力和发现对象执行完整 Schema 校验，保留源码位置和结构化核心错误；
- [x] 为六目标生成确定性 SBOM、构建元数据、逐目标锁和构建来源证明；
- [x] 先验验证发行主体与签名者，隔离安装凭据，并保护草稿、预发布、稳定版及 latest 状态转换。

## 0.7.0：大型工作区与性能

- 优化多项目解析、包图缓存、并发项目锁和失败取消；
- 支持批量 check/test/build、`--keep-going`与稳定 JSON Lines 进度；
- 建立 100 个以上项目的压力与增量回归。

## 0.7.1：生态与最终稳定化

- 完成官方生态项目的 resolve/check/test/build/audit/pack/vendor 兼容矩阵；
- 完善注册表只读元数据、工作区共享锁和离线/vendor 恢复；
- 稳定工程协议 Schema、版本协商与 LSP 多工作区调用；
- 完成 P0/P1 清零和发布后安装验证。
